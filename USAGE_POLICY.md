# Usage Policy

This project is for approval-gated side effects and agent action governance.

Use it only for workflows where:

- the account owner has authorized the action
- a human approval gate is enforced before privileged execution
- the broker remains the source of truth for proposal and execution state
- audit history is preserved

Do not use this project for:

- spam
- bulk sending
- unsolicited commercial email
- bypassing service limits
- hiding automation from users
- collecting or storing Gmail or Google credentials in adapters
- implying Google or Gmail affiliation, endorsement, or sponsorship

For personal real Gmail usage, prefer:

- Gmail API for draft creation
- Gmail API for non-native send-now flows where acceptable
- browser execution only when native Gmail schedule-send behavior is required
- an OpenClaw-controlled Chrome session/profile for browser execution
